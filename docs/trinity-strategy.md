# 三位一体合约交易策略 (Trinity Strategy)

> **PA + SMC + Wyckoff 三体系融合**  
> 编制日期: 2026-05-28  
> 用途: 妖币交易系统第四模块核心策略

---

## 一、融合哲学：三层分析框架

### 1.1 为什么要融合？

三种体系各有盲区，互补形成完整拼图：

| 体系 | 优势 | 盲区 | 在框架中的角色 |
|------|------|------|---------------|
| **Wyckoff** | 大周期结构判断，机构意图识别 | 入场精度粗糙，不适合短线 | **结构层** — 回答"该不该做" |
| **SMC** | 流动性分析精准，价位定位明确 | 缺乏宏观阶段背景 | **机构层** — 回答"在哪里做" |
| **PA (Brooks)** | 入场时机精确，K线信号完备 | 缺乏大局观和流动性视角 | **执行层** — 回答"什么时候做" |

### 1.2 三层分析架构

```
┌──────────────────────────────────────────────────────┐
│  第一层：Wyckoff 结构层  (日线/4H)                    │
│  判断市场宏观阶段：吸筹/上涨/派发/下跌               │
│  → 输出：多空偏向 (BULL/NEUTRAL/BEAR)                 │
├──────────────────────────────────────────────────────┤
│  第二层：SMC 机构层  (4H/1H)                         │
│  识别流动性区域、订单块、结构转变                     │
│  → 输出：关键价位 (POI) + 流动性状态                  │
├──────────────────────────────────────────────────────┤
│  第三层：PA 执行层  (1H/15min)                        │
│  H2/L2 入场信号 + 信号K线质量 + 始终在场方向          │
│  → 输出：入场/止损/目标                              │
└──────────────────────────────────────────────────────┘
```

### 1.3 核心原则

> **三层一致则交易，矛盾则观望。**
> 威科夫说牛市吸筹 → SMC找到看涨OB → PA发出H2买入信号 = 高胜率交易  
> 威科夫说派发阶段 → 即使PA有完美H2 → 不做多

## 二、第一层：Wyckoff 结构判断

### 2.1 市场八阶段模型

```
┌──────────────────────────────────────────┐
│  熊 → 吸筹 → 牛 → 再吸筹 → 派发 → 再派发  │
└──────────────────────────────────────────┘
```

| 阶段 | 威科夫事件序列 | 多空偏向 | 量化关键特征 |
|------|---------------|---------|-------------|
| **吸筹 A-B** | SC→AR→ST(s) | NEUTRAL | 波动率下降，成交量底部放大 |
| **吸筹 C** | Spring/震仓 | BULL | 假跌破后快速收回，低量确认 |
| **吸筹 D-E** | SOS→LPS→BUEC | BULL(强) | 突破阻力，回踩不破 |
| **上涨趋势** | HH/HL序列 | BULL | 更高高点和更高低点 |
| **派发 A-B** | BC→AR→ST(s) | NEUTRAL | 波动率下降，成交量顶部放大 |
| **派发 C** | UTAD/诱多 | BEAR | 假突破后快速回落，低量确认 |
| **派发 D-E** | SOW→LPSY | BEAR(强) | 跌破支撑，反弹不破 |
| **下跌趋势** | LH/LL序列 | BEAR | 更低高点和更低低点 |

### 2.2 量化检测规则

| 特征 | 检测逻辑 | 权重 |
|------|---------|------|
| **高潮K线检测** | K线实体 > 1.5× 近期均值 + 成交量 > 2× 均值 | SC/BC信号 |
| **AR区间确认** | 从高潮K线算起，高点/低点形成区间上/下边界 | 参考位 |
| **二次测试确认** | 回调至SC低点/BC高点区域，成交量<首次70% | ST确认 |
| **Spring检测** | 跌破区间低点后2根K线内收回区间内 | 看涨信号 |
| **UTAD检测** | 突破区间高点后2根K线内回到区间内 | 看跌信号 |
| **SOS确认** | 突破区间阻力 + 放量 + 连续2根K线收于阻力上方 | 趋势确认 |
| **SOW确认** | 跌破区间支撑 + 放量 + 连续2根K线收于支撑下方 | 趋势确认 |
| **趋势状态** | 连续3+个HH/HL(Bull)或LH/LL(Bear) | 趋势确认 |

### 2.3 Wyckoff 层输出

```python
WYCKOFF_SIGNAL = {
    "phase": "ACCUMULATION_C" | "MARKUP" | "DISTRIBUTION_C" | "MARKDOWN" | "NEUTRAL",
    "bias": "BULL" | "BEAR" | "NEUTRAL",
    "confidence": 0-100,  # 结构置信度
    "key_events": ["Spring", "ST_pass"],
    "range_high": float,  # 区间上边界
    "range_low": float,   # 区间下边界
}
```

## 三、第二层：SMC 机构层

### 3.1 核心概念映射

| SMC概念 | 对应Wyckoff概念 | 含义 |
|---------|----------------|------|
| **SSL猎杀** | Spring | 收割多头止损后反转上涨 |
| **BSL猎杀** | UTAD | 收割空头止损后反转下跌 |
| **看涨OB** | LPS + 需求区 | 上涨前机构最后的买入区域 |
| **看跌OB** | LPSY + 供应区 | 下跌前机构最后的卖出区域 |
| **CHoCH/MSS** | SOS/SOW | 结构转变确认 |
| **FVG** | — | 价格失衡区，可作为目标 |
| **OTE (62-79%)** | — | 入场精确定位 |

### 3.2 量化检测规则

| 特征 | 检测逻辑 | 权重 |
|------|---------|------|
| **市场结构** | HH/HL序列 = 看涨; LH/LL序列 = 看跌 | 基础 |
| **BOS** | 突破最后一个相关高/低点（延续） | 趋势确认 |
| **CHoCH** | 首次突破最后一个HL/LH，K线温和 | 改变警告 |
| **MSS** | 同CHoCH但使用大实体K线猛烈突破 | 高置信改变 |
| **SSL猎杀** | 跌破前低后反转，2根K线收回 | 看涨触发 |
| **BSL猎杀** | 涨破前高后反转，2根K线收回 | 看跌触发 |
| **看涨OB** | 上涨前最后一根阴线（需含BOS+位移） | 做多POI |
| **看跌OB** | 下跌前最后一根阳线（需含BOS+位移） | 做空POI |
| **FVG** | K线1影线与K线3影线不重叠（三根K线） | 目标/POI |
| **OTE区间** | 冲动走势的62%-79%斐波那契回撤 | 入场优化 |

### 3.3 SMC 层输出

```python
SMC_SIGNAL = {
    "structure": "BULLISH" | "BEARISH" | "RANGE",
    "bos_count": int,      # 近期BOS次数
    "choch": bool,         # 是否有CHoCH
    "mss": bool,           # 是否有MSS
    "liquidity_sweep": "SSL" | "BSL" | None,  # 最近流动性猎杀
    "order_block": {
        "type": "BULL" | "BEAR" | None,
        "proximal": float,  # 近端线
        "distal": float,    # 远端线
        "quality": 0-100    # OB质量评分
    },
    "fvg": {
        "type": "BULL" | "BEAR" | None,
        "top": float,
        "bottom": float
    },
    "ote_zone": {
        "low": float,       # 62%
        "sweet": float,     # 70.5%
        "high": float       # 79%
    },
    "poi_list": [           # 所有兴趣点
        {"type": "OB", "level": float, "direction": "LONG"},
        {"type": "FVG", "level": float, "direction": "LONG"}
    ]
}
```

## 四、第三层：PA (Brooks) 执行层

### 4.1 核心概念

| PA概念 | 含义 | 应用 |
|--------|------|------|
| **始终在场方向** | 被迫持仓时的方向选择 | 只在此方向交易 |
| **H2/L2** | 双腿回调后的二次入场 | 主要入场信号 |
| **信号K线质量** | 实体/影线/收盘位置评分 | 过滤差信号 |
| **20 EMA** | 动态支撑/阻力 | 回调入场参考 |
| **高潮K线** | 异常大的K线(1.5-2×均值) | 耗尽警告 |
| **铁丝网** | 3根以上高度重叠K线 | 不交易 |
| **测量移动** | 腿1≈腿2 | 波段目标 |

### 4.2 量化检测规则

| 特征 | 检测逻辑 | 权重 |
|------|---------|------|
| **始终在场方向** | EMA斜率+HL序列+趋势K线占比 | 核心 |
| **回调腿数计数** | 逆趋势方向的摆动次数 | H1/H2/H3 |
| **H2信号** | 第二腿回调+信号K线高点突破前K线 | 最高胜率入场 |
| **L2信号** | 第二腿反弹+信号K线低点跌破前K线 | 最高胜率入场 |
| **信号K线评分** | 实体占比(0-40) + 收盘位置(0-30) + 影线(0-20) + 大小(0-10) | ≥60通过 |
| **20EMA测试** | 价格回调触及或接近20EMA | 入场确认 |
| **铁丝网检测** | 连续3+根K线重叠度>70% + 小实体 | 暂停交易 |
| **趋势强度** | 连续顺势K线数 + 回调深度 + EMA距离 | 1-5级 |

### 4.3 PA 层输出

```python
PA_SIGNAL = {
    "always_in": "LONG" | "SHORT" | "NEUTRAL",
    "trend_strength": 1-5,      # 趋势强度
    "callback_legs": 1|2|3|4,   # 当前回调腿数
    "h2_ready": bool,           # H2信号就绪
    "l2_ready": bool,           # L2信号就绪
    "signal_bar_quality": 0-100,# 信号K线质量评分
    "ema_position": "ABOVE" | "BELOW" | "AT",  # 价格相对20EMA
    "is_barbwire": bool,        # 是否为铁丝网
    "climax_warning": bool,     # 高潮耗尽警告
    "measured_move_target": float,  # 测量移动目标
}
```

## 六、风险管理

### 6.1 止损规则

```
多头止损 = max(
    SMC看涨OB远端线下方1%,
    PA信号K线低点下方1%,
    Wyckoff Spring低点下方1%
)

空头止损 = min(
    SMC看跌OB远端线上方1%,
    PA信号K线高点上1%,
    Wyckoff UTAD高点上1%
)
```

### 6.2 获利目标

| 优先级 | 目标 | 平仓比例 |
|--------|------|---------|
| TP1 | 初始风险的1×（最近摆动点） | 平50% |
| TP2 | FVG/测量移动目标 | 平30%，止损移成本 |
| TP3 | Wyckoff区间对侧边界 | 平20%（追踪止损） |

### 6.3 仓位管理

| 规则 | 数值 |
|------|------|
| 单笔最大风险 | 账户的2% |
| 最大同时持仓 | 2个币种 |
| 日最大亏损 | 账户的6% |
| 连续亏损停止 | 3笔 → 当天停止 |

### 6.4 仓位计算

```python
position_size = (account * risk_pct) / (stop_distance_pct * leverage)
# 例: $100 × 2% / (1% × 5×) = $40 仓位
```

## 七、完整交易流程

```
1. [WYCKOFF] 日线分析 → 确定宏观阶段
   ├─ 吸筹/上涨 → 只做多
   ├─ 派发/下跌 → 只做空
   └─ 中性/不确定 → 不交易或仅短线

2. [WYCKOFF] 4H分析 → 确认二次结构
   ├─ Spring通过? UTAD通过?
   ├─ SOS/SOW确认?
   └─ 记录区间高/低边界

3. [SMC] 1H分析 → 寻找兴趣点
   ├─ 结构方向是否与Wyckoff一致?
   ├─ 是否有流动性猎杀(SSL/BSL)?
   ├─ 定位OB + FVG
   ├─ 计算OTE区间
   └─ 输出POI列表

4. [SMC] 15min → 细化POI
   ├─ 低时间框架内是否有更精确的OB?
   ├─ 价格是否接近OTE?
   └─ 等待价格接近POI

5. [PA] 执行时间框架 → 等待入场
   ├─ H2/L2信号是否形成?
   ├─ 信号K线质量是否≥60?
   ├─ 是否为铁丝网?(是=跳过)
   ├─ 是否有高潮警告?(有=谨慎)
   └─ 始终在场方向是否匹配?

6. [执行] 入场
   ├─ 计算仓位大小
   ├─ 设置止损(三层中最保守的)
   ├─ 设置TP1/TP2/TP3
   └─ 记录交易日志

7. [管理] 持仓中
   ├─ TP1达成 → 平50%，止损移成本
   ├─ 追踪剩余仓位
   └─ 新反向结构出现 → 全部平仓
```

## 九、加密货币适配调整

| 传统市场特征 | 加密货币差异 | 调整方案 |
|-------------|-------------|---------|
| 4小时交易时段 | 24/7交易 | Killzone改为UTC关键时段(欧美亚盘开盘) |
| 日线收盘明确 | 无真正收盘 | 用UTC 00:00作为"日收盘" |
| 波动率适中 | 极端波动 | ATR系数从1×调整为1.5-2×止损 |
| 无资金费率 | 永续合约有资金费率 | 做多时检查资金费率(过高不做) |
| 成交量真实 | 交易所成交量可能注水 | 用多家交易所成交量均值或币安成交量 |
| Tick精度高 | 价差较大 | 止损/入场缓冲区适当放宽 |
